# 眼球追踪

## 功能概述

idle 状态下，桌宠的瞳孔跟随鼠标位置移动。鼠标离开窗口时瞳孔平滑回到中心。

## 算法

```
1. 获取鼠标在窗口坐标系中的位置 (mouseX, mouseY)
2. 从 Rendering.getBounds() 获取头部中心坐标 (headCX, headCY)
3. 计算鼠标相对于头部中心的偏移:
     dx = mouseX - headCX
     dy = mouseY - headCY
4. 映射到眼珠移动范围:
     rawPX = dx / sensitivityRadius
     rawPY = dy / sensitivityRadius
5. 约束范围:
     水平: ±1.5 格 (clamp)
     垂直: ±1.0 格
6. 平滑跟随 (lerp 速率 0.28):
     eyePupilX += (targetX - eyePupilX) × 0.28
     eyePupilY += (targetY - eyePupilY) × 0.28
```

## 约束条件

| 参数 | 值 | 说明 |
|------|-----|------|
| 水平范围 | ±1.5 格 | 眼珠不超出头部边界 |
| 垂直范围 | ±1.0 格 | 眼珠不超出头部上下边界 |
| 灵敏度半径 | max(petSize × 1.2, 60px) | 鼠标在此范围内瞳孔达到最大偏移 |
| lerp 速率 | 0.28 | 平滑跟随系数 |
| 激活条件 | `M.FSM.state === 'idle'` | 仅空闲时激活 |

## 特殊情况

| 情况 | 行为 |
|------|------|
| 鼠标在窗口内 | 瞳孔跟随鼠标，lerp 平滑过渡 |
| 鼠标离开窗口 | 瞳孔缓动回到中心 (0, 0) |
| 非 idle 状态 | 瞳孔强制归中，不追踪 |

## 实现位置

`renderer/interaction/eye-tracking.js`，通过 `M.EyeTracking.update()` 每帧调用。

## 已知问题

快速移动鼠标时瞳孔可能有轻微视觉抖动。根因是一阶 lerp 在低帧率下不够平滑。影响轻微，优先度低。
