# 眼球追踪

## 功能概述

idle 状态下，桌宠的瞳孔跟随鼠标位置移动。鼠标离开窗口时瞳孔平滑回到中心。

v3.7.1 重写，核心改进：基于**角色头部中心**而非窗口中心计算偏移。

## 算法

```
1. 获取鼠标在窗口坐标系中的位置 (mouseX, mouseY)
2. 计算头部中心在窗口中的坐标:
     headCX = petCX
     headCY = petCY - (petSize / 2) + (headHeight / 2)
3. 计算鼠标相对于头部中心的偏移:
     dx = mouseX - headCX
     dy = mouseY - headCY
4. 映射到眼珠移动范围:
     水平: ±2 格 (每格 = petSize / 16)
     垂直: ±1 格
5. 平滑跟随 (lerp 速率 0.28):
     eyeOffsetX += (targetX - eyeOffsetX) × 0.28
     eyeOffsetY += (targetY - eyeOffsetY) × 0.28
6. 约束在头部区域内 (cols 4-11, rows 0-6)
```

## 约束条件

| 参数 | 值 | 说明 |
|------|-----|------|
| 水平范围 | ±2 格 | 眼珠不超出头部左右边界 |
| 垂直范围 | ±1 格 | 眼珠不超出头部上下边界 |
| 头部区域 | cols 4-11, rows 0-6 | 16×20 网格中的头部范围 |
| lerp 速率 | 0.28 | 平滑跟随系数 |
| 激活条件 | `M.FSM.state === 'idle'` | 仅空闲时激活 |

## 特殊情况

| 情况 | 行为 |
|------|------|
| 鼠标在窗口内 | 瞳孔跟随鼠标，lerp 平滑过渡 |
| 鼠标离开窗口 | 瞳孔缓动回到中心 (0, 0) |
| 非 idle 状态 | 瞳孔强制归中，不追踪 |
| 低功耗模式 (15fps) | 仍运行追踪逻辑但更新频率降低 |

## 已知问题

快速移动鼠标时瞳孔可能有轻微视觉抖动。根因是一阶 lerp 在低帧率下不够平滑。修复方向：二阶平滑或自适应 lerp 系数。当前影响轻微，暂未修复。
