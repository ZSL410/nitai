# 听歌模式

## 功能概述

听歌模式（listening 状态）让桌宠"感知"你正在听音乐，自动或手动戴上耳机摇摆。

## 触发方式

### 手动触发

在聊天框（Enter）输入以下关键词：

- `听歌`
- `music`
- `耳机`

### 自动检测

`main/music-detector.js` 后台每 3 秒执行 `tasklist` 扫描系统进程列表，匹配配置的播放器名。

默认监控的播放器：

```json
["spotify", "qqmusic", "SodaMusic", "cloudmusic", "netease", "kwmusic", "kugou"]
```

可在 `config.json` 中修改：

```json
{
  "musicDetect": true,
  "musicCheckInterval": 3,
  "musicPlayers": ["spotify", "qqmusic", "..."]
}
```

### 差异

| 触发方式 | 自动退出 |
|----------|----------|
| 手动输入"听歌" | ❌ 需手动输入"停止"退出 |
| 自动检测到播放器 | ✅ 播放器关闭后自动回到 idle |

## 视觉效果

| 元素 | 描述 |
|------|------|
| 耳机 | `pixel-character.js` 中 `drawHeadphones()` 绘制头梁 + 耳罩（像素风格） |
| 眼型 | `closed_arch`（开心闭眼弧线） |
| 腮红 | ✅ 粉色腮红 |
| 身体 | 左右摇摆（正弦波驱动） |
| 手臂 | 轻微摆动 |
| 粒子 | 每 600ms 飘出 1 个音符 ♪ |

## 退出

- 手动触发 → 输入 `停止` / `stop`
- 自动触发 → 播放器进程从列表消失后自动退出
- 输入 `停止` 可同时退出 listening 和 bongo 状态

## 技术实现

```
主进程: music-detector.js
  → child_process.exec('tasklist')
  → 匹配 musicPlayers 列表
  → IPC 'music-playing' / 'music-stopped'

渲染进程: interaction/music-detect.js
  → 接收 IPC
  → M.FSM.transitionTo('listening') / transitionTo('idle')

FSM 状态: fsm/states/listening.js
  → expression: 'listening' (closed_arch + none + cheek)
  → update: 身体摇摆 + 音符粒子定时发射
```
