# 术语表

> 拟态项目专用术语定义。保持统一命名，避免沟通歧义。

| 术语 | 英文 | 定义 |
|------|------|------|
| **拟态** | nitai / Mimic | 项目名称。取自动物学 mimicry（拟态），意为融入桌面环境的小生物。也是全局命名空间 `window.Mimic`。 |
| **桌宠** | desktop pet | 运行在桌面上的虚拟宠物角色，16×20 像素网格小人。 |
| **晓风** | xiaofeng | 晓风智能体主项目，拟态的后端。运行在 `http://127.0.0.1:5001`。 |
| **世界坐标** | world coordinates | 角色在屏幕坐标系中的位置（`worldX`, `worldY`）。v1.0.0 核心概念——窗口跟随世界坐标移动。 |
| **FSM** | Finite State Machine | 有限状态机。驱动角色行为和表情的核心引擎（`behavior/state-machine.js`）。 |
| **自主AI** | autonomous AI | 层次化行为决策系统（`behavior/ai-controller.js`）。每 3 秒决策闲逛/好奇心/睡眠/微动作。 |
| **召唤系统** | summon system | 点击桌面任意位置让角色走过去的机制。三通道触发：右键菜单/三击/文字指令。 |
| **生命感** | life-sense | 始终运行的底层动画：呼吸（3s 正弦波）+ 眨眼（2-6s 随机）+ 微晃动（5s 正弦波）。 |
| **damp** | exponential damping | 帧率无关的指数衰减缓动算法。公式：`a + (b-a)*(1-e^(-λ*dt))`。用于角色行走。 |
| **闲逛** | wander | 角色自主走到随机屏幕位置的行为。由 AI 控制器以 35% 概率触发。 |
| **气泡** | bubble / speech bubble | 角色上方的对话框。消息队列系统，从嘴巴上方"长"出，带箭头尾巴。 |
| **粒子** | particle | 飘散的特效元素。4 种：心形(❤)、星形(✦)、火花(·)、音符(♪)。 |
| **表情** | expression | 眼型 + 嘴型 + 腮红的组合。数据驱动，12 种预定义组合。 |
| **眼型** | eye type | 眼睛的像素绘制模式。8 种：round / arch / wide / closed / closed_arch / sparkle / half_closed / heart。 |
| **嘴型** | mouth type | 嘴巴的像素绘制模式。5 种：none / smile / open / grin / o。 |
| **穿透模式** | passthrough | 鼠标事件穿透到下层窗口（`setIgnoreMouseEvents`）。`Ctrl+Shift+P` 切换。 |
| **IPC** | Inter-Process Communication | Electron 主进程与渲染进程通信。拟态使用扁平参数传递。 |
| **Web Audio** | Web Audio API | 浏览器内置音频合成 API。拟态用它合成 4 种音效（chirp/boing/nom/footstep），无需外部文件。 |
| **消息队列** | message queue | 气泡系统的多条消息顺序显示机制。最多缓存 8 条。 |
| **音乐检测** | music detection | 后台 3s 轮询系统进程列表，检测 Spotify/QQ音乐/网易云等播放器。 |
| **Pointer Capture** | pointer capture | 指针锁定机制，用于可靠的窗口拖拽。WSL2 下退化为 document 级追踪。 |
| **召唤覆盖层** | summon overlay | 临时全屏透明 BrowserWindow，捕获一次点击后立即销毁。用于召唤系统。 |
