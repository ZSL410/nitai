# v3.7.1 — Phase 1 交互体验优化

> 发布日期：2026-06-27
> 版本号：`3.7.1`
> 变更类型：功能增强 + 性能优化

## 概述

本次更新聚焦三项交互体验优化：增强眼球跟随鼠标追踪精度、完善点击部位状态反馈、引入低功耗空闲模式降低资源占用。

---

## 详细变更

### 1. 眼球跟随鼠标增强

**涉及文件**：`renderer/interaction/eye-tracking.js`、`renderer/rendering/pixel-character.js`

**变更前 (v2.x)**：
- 基于窗口中心计算鼠标偏移量
- 眼珠移动范围 ±1 格

**变更后 (v3.7.1)**：
- 基于**角色头部中心**（grid col 7.5, row 3.0）计算偏移量，追踪更自然
- 眼珠移动范围扩展至水平 ±1.5 格、垂直 ±1.0 格（严格限制在头部 cols 4-11 / rows 0-6 区域内）
- 渲染层支持 ±2×±1 格像素位移（`Math.round` 取整）
- 平滑跟随速率提升至 0.28，响应更快
- 灵敏度半径 = `max(petSize × 1.2, 60px)`，与宠物体型自适应

**关键实现**：
- `eye-tracking.js`：新增 `headCX`/`headCY` 计算（基于 canvas 坐标），`PUPIL_MAX_DX=1.5`/`PUPIL_MAX_DY=1.0` 边界常量
- `pixel-character.js`：`px` clamp 从 `[-1,1]` 扩展至 `[-2,2]`

### 2. 点击部位反馈升级

**涉及文件**：`renderer/interaction/click.js`

**变更前 (v2.x)**：
- 点击身体仅显示气泡 + 小弹跳，**不触发 FSM 状态切换**

**变更后 (v3.7.1)**：
- 点击身体 → 触发 `surprised` 状态（wide 眼睛 + open 嘴巴 + 抖动衰减 + boing 音效 + 火花粒子），**1.5 秒后自动恢复 idle**
- 点击头部 → 原有 `happy` 反馈保留（弹跳 + 微笑 + chirp + 心形粒子，1.5s）
- 双击 → 原有旋转弹跳保留（2.5s）
- 增加火花粒子（`Particles.burst('spark', 4)`）于身体点击

**状态转换安全性**：仅在当前状态为 `idle` 时触发 FSM 转换，防止覆盖 active 状态。

### 3. 低功耗空闲模式

**涉及文件**：`renderer/app.js`

**新增行为**：
- 角色处于 `idle` 状态且**连续 30 秒无鼠标移动**（`mousemove` 事件）→ 绘制帧率降至 **15fps**
- 帧跳过机制：通过 `_idleFrameCounter` 跟踪，每 `_idleLowPowerSkip`（=4）帧绘制 1 帧
- 鼠标移动（`mousemove` 触发 `lastActivity` 更新）→ **立即恢复 60fps** 全速绘制
- 退出低功耗模式时重置帧计数器
- 切换体型 → `mousemove` 事件自动刷新 `lastActivity`，无需额外 hook

**性能收益**：
- 空闲时 CPU/GPU 占用降低约 75%（60fps → 15fps = 每秒绘制次数减少 75%）
- 不影响交互体验：鼠标移动即时唤醒

**关键实现**：
```javascript
// 状态变量（挂载在 M 命名空间）
M._idleLowPower = false;          // 是否处于低功耗模式
M._idleLowPowerThreshold = 30000; // 30 秒阈值
M._idleLowPowerFps = 15;          // 目标帧率
M._idleLowPowerSkip = 4;          // 每 N 帧绘制 1 帧
```

## 兼容性说明

- **向后兼容**：所有 v2.5.0 功能完整保留（文件拖放、气泡、右键菜单、穿透、听歌、音乐检测等）
- **无破坏性变更**：API 接口、IPC 通道、配置文件格式均不变
- **FSM 行为**：`surprised` 状态退出时自动重置肢体动画（arm/leg 归零），不影响后续状态切换
