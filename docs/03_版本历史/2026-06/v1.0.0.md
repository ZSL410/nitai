### 📐 坐标系统重构（核心）

**问题**：v2.0 架构重写时设计了动态布局公式（`petCX=winW/2, petCY=winH/2`），但 `pixel-character.js`、`eye-tracking.js`、`click.js` 中各自独立计算 `gx`/`gy`/`cellSize`，且存在细微不一致：
- `click.js` 使用 `M.Anim.bobOffset` 而非 `M.Anim._smoothBob`，导致点击命中测试与视觉位置不匹配
- `eye-tracking.js` 和 `pixel-character.js` 的 `cellSize` 最小值不一致（2 vs 承自 old code）
- 4 处代码各自维护相同的计算逻辑

**解决方案**：在 `renderer/layout.js` 中新增 `getPetBounds()` 函数，作为所有坐标计算的**单一真相来源**（Single Source of Truth）。

#### getPetBounds() 返回结构

```javascript
{
  centerX, centerY,          // 窗口中心 = 角色中心
  petSize, scale, effectiveSize,
  cellSize, totalW, totalH,  // 网格像素尺寸
  gx, gy, bobY,              // 网格原点 + 平滑弹跳偏移
  headCX, headCY, headTop,   // 头部位置
  headLeft, headRight,       // 头部水平边界
  mouthX, mouthY,            // 嘴部位置（气泡箭头指向）
  gridW: 16, gridH: 20,      // 网格单元格数
}
```

#### 重构范围

| 文件 | 变更 |
|------|------|
| `layout.js` | 新增 `getPetBounds()` + 网格常量（GW/GH/MOUTH_GRID/HEAD_COLS等）|
| `pixel-character.js` | 移除内部 `cellSize`/`gx`/`gy`/`totalW`/`totalH` 计算，改为 `getPetBounds()` |
| `eye-tracking.js` | 移除内部 `gx`/`gy`/`headCX`/`headCY` 计算，改为 `getPetBounds().headCX/headCY` |

| `bubble.js` | 嘴部位置优先从 `getPetBounds()` 获取 |

## 修复的 Bug

### click.js: bobOffset vs _smoothBob 不一致

**Bug**: `click.js` 的 `hitTest()` 使用 `M.Anim.bobOffset`（瞬时值）计算 `gy`，而绘制使用 `_smoothBob`（lerp 平滑值）。当角色弹跳时，点击判定位置与视觉位置不匹配。

**修复**: 改为使用 `getPetBounds().gy`，其中 `gy` 基于 `_smoothBob` 计算，与绘制保持一致。
