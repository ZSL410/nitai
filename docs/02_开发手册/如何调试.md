# 如何调试

## 打开 DevTools

### 方式一：代码触发

在渲染进程代码中调用：

```javascript
const { remote } = require('electron');
remote.getCurrentWindow().webContents.openDevTools({ mode: 'detach' });
```

### 方式二：主进程启动参数

临时在 `main.js` 中添加：

```javascript
win.webContents.openDevTools({ mode: 'detach' });
```

### 方式三：快捷键（未实现）

当前版本没有内置 DevTools 快捷键。可自行在 `mimic.js` 的 keydown 事件中添加。

## 查看日志

### 主进程日志

主进程 `console.log` 输出到启动桌宠的命令行终端。关键日志：

```
[main] 拟态 Desktop Pet ready
[main] global shortcut: passthrough → ON
```

### 渲染进程日志

打开 DevTools 后在 Console 面板查看。关键日志：

```
[mimic] global namespace initialized v3.7.1
[fsm/core] FSM engine initialized
[fsm/expressions] registered 11 expressions
[layout] module initialized
[fsm] idle → happy {"trigger":"head"}
[layout] pet=80 bubbleH=24 overhead=35 => win=120x150 petC=(60,75)
```

## 调试 IPC 通信

在 DevTools Console 中手动发送 IPC：

```javascript
// 测试窗口拖拽
window.Mimic.ipc.send('window-drag', 10, 0);

// 测试穿透切换
window.Mimic.ipc.send('toggle-passthrough');

// 查看当前状态
window.Mimic.FSM.state;
window.Mimic.FSM.getExpression();
```

## 调试 FSM 状态

```javascript
// 手动切换状态
window.Mimic.FSM.transitionTo('happy');
window.Mimic.FSM.transitionTo('bongo');
window.Mimic.FSM.transitionTo('listening', { autoDetected: true });

// 查看当前所有状态
console.log({
  state: M.FSM.state,
  prevState: M.FSM.prevState,
  stateTimer: M.FSM.stateTimer,
  expression: M.FSM.getExpression()
});
```

## 调试布局

```javascript
// 查看布局参数
const M = window.Mimic;
console.log({
  petSize: M.petSize,
  winW: M.winW, winH: M.winH,
  petCX: M.petCX, petCY: M.petCY,
  petTop: M.petCY - M.petSize/2,
  bubbleH: M.currentBubbleH
});

// 手动触发布局重算
M.Layout.updateWindowSizeAndLayout('测试气泡文本');
```

## 调试粒子

```javascript
// 手动发射粒子
const M = window.Mimic;
if (M.Particles) {
  M.Particles.emit('heart', M.petCX, M.petCY - 20, 5);
  M.Particles.emit('note', M.petCX + 20, M.petCY - 30, 3);
}
```

## 常见调试场景

| 问题 | 调试方法 |
|------|----------|
| 角色位置偏上 | 检查 `petCX/petCY` 与实际绘制坐标的偏移 |
| 气泡箭头不对齐 | 检查 `bubbleEl.style.top` 计算值 |
| 眼球不跟随 | 检查 `M.FSM.state === 'idle'` 是否为 true |
| 音效不播放 | 检查 AudioContext 状态（可能被浏览器策略阻塞） |
| 文件拖放无反应 | 检查 `targetDir` 路径是否存在 |

## 性能分析

在 DevTools Performance 面板录制动画循环：

1. 打开 DevTools → Performance 标签
2. 点击录制
3. 操作桌宠（拖拽/点击/切换体型）
4. 停止录制 → 查看帧率和调用栈
