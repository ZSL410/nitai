# 如何添加新表情

## 概述

表情系统是数据驱动的。添加新表情只需修改 2 个文件，如涉及新眼型/嘴型则需额外修改渲染层。

## 步骤一：定义表情（expressions.js）

编辑 `renderer/fsm/expressions.js`，在 `EXPRESSIONS` 对象中添加新条目：

```javascript
const EXPRESSIONS = {
  // ... 已有表情 ...
  
  excited: {
    name: '兴奋',
    eyes: 'sparkle',      // 使用已有眼型
    mouth: 'grin',        // 使用已有嘴型
    cheek: true,          // 显示腮红
  },
};
```

## 步骤二：在状态中使用

在对应 FSM 状态的 handler 中引用：

```javascript
// fsm/states/happy.js
const handler = {
  expression: function() {
    // 根据上下文动态返回
    if (this.context.intensity === 'high') {
      return 'excited';  // 新表情
    }
    return 'happy';
  },
  // ...
};
```

静态表情直接返回字符串：

```javascript
expression: 'excited',
```

## 步骤三（可选）：添加新眼型

如果现有 7 种眼型不够用，在 `renderer/rendering/pixel-character.js` 中添加绘制函数。

### 眼型绘制规范

```javascript
// 在 drawEyes 函数中添加新的 case
function drawEyes(ctx, eyeType, cx, cy, gridSize, offsetX, offsetY) {
  switch (eyeType) {
    case 'heart':  // 新眼型：心形眼
      // 左眼位置
      const lx = cx + (offsetX || 0) - 4 * gridSize;
      const ly = cy + (offsetY || 0) - 10 * gridSize;
      // 右眼位置
      const rx = cx + (offsetX || 0) + 2 * gridSize;
      const ry = cy + (offsetY || 0) - 10 * gridSize;
      
      // 绘制心形（两个像素弧线组成心形上半部 + 三角形下半部）
      // ... 返回实际占用的像素坐标
      break;
  }
}
```

### 关键约束

- 所有坐标必须基于 `petCX`/`petCY`/`petSize` 动态计算
- **禁止硬编码像素值**（参见 `02_开发手册/常见问题.md` → 坐标偏上问题）
- 眼型绘制函数签名需与现有眼型一致

## 步骤四（可选）：添加新嘴型

同眼型，在 `pixel-character.js` 的 `drawMouth` 函数中添加。

## 步骤五：在 index.html 中无需额外加载

表情定义在 `expressions.js` 中，已在 `index.html` 加载链中，无需新增 `<script>` 标签。

## 验证

1. 启动桌宠
2. 打开 DevTools Console
3. 手动触发新表情验证渲染：

```javascript
// 临时修改 FSM 返回新表情
const orig = M.FSM.getExpression;
M.FSM.getExpression = () => M.EXPRESSIONS.excited;
M.Rendering.draw();

// 恢复
M.FSM.getExpression = orig;
```

4. 在三种体型（50/80/110）下验证表情居中、比例正确
