# 常见问题

## 启动问题

**Q: 启动后窗口不显示？**
A: 检查是否在 WSL2 中运行。WSL2 不支持 GUI，请在 Windows 原生命令行中运行 `npm start`。

**Q: 提示 `electron: command not found`？**
A: 依赖未安装。运行 `cd D:\nitai\拟态开发版 && npm install`。

**Q: `npm start` 报错？**
A: 确保 Node.js ≥ 18 且 `node_modules/` 完整。如有问题尝试删除 `node_modules/` 后重新 `npm install`。

## 显示问题

**Q: 角色位置偏上，不在窗口中央？** 🐛
A: 已知问题。部分肢体动画坐标使用硬编码偏移。参见下方"坐标硬编码问题"。计划 v3.8.0 / 独立后 v1.0.0 修复。

**Q: 气泡箭头遮挡角色？** 🐛
A: 坐标硬编码问题的连锁反应。当角色偏上时，气泡定位计算亦偏移，箭头指向不准。相关：在非 80px 体型下尤为明显。

**Q: 眼球追踪有抖动？**
A: 一阶 lerp 平滑（速率 0.28）在快速移动或低帧率时有轻微抖动。v3.7.1 已改善（改为基于头部中心计算偏移），但未完全消除。影响轻微，优先度低。

**Q: 三种体型下角色表现不一致？**
A: 坐标硬编码导致。50px 体型下角色偏下，110px 下偏上。仅 80px（默认）下基本居中。

## 功能问题

**Q: 文件拖放没反应？**
A: 检查：
1. `config.json` 中 `targetDir` 路径是否存在
2. 拖放的文件是否可读
3. DevTools Console 中是否有报错

**Q: 听歌模式不自动触发？**
A: 检查：
1. `config.json` 中 `musicDetect` 是否为 `true`
2. 播放器名称是否在 `musicPlayers` 列表中
3. 播放器进程名是否与配置完全一致（区分大小写）

**Q: 手动输入"听歌"后，音乐停了它还在听歌？**
A: 这是设计如此。手动触发的 listening 不会自动退出，需要输入"停止"或"stop"退出。仅自动检测触发的 listening 会在音乐停止后自动退出。

**Q: 穿透模式下无法操作桌宠？**
A: `Ctrl+Shift+P` 全局快捷键（主进程注册）在穿透模式下仍然可用。也可通过系统托盘菜单关闭穿透。

**Q: Ctrl+Q 无法退出？**
A: 窗口需要有焦点才能接收键盘事件。如果窗口失去焦点，使用系统托盘菜单退出。

## ⚠️ 坐标硬编码问题

**严重程度**：中 | **计划修复**：v3.8.0 / 独立后 v1.0.0

### 问题描述

v2.0 架构重写时，布局公式设计为动态计算（`petCX=winW/2`, `petCY=winH/2`），但 `renderer/rendering/pixel-character.js` 中的肢体动画端点仍使用旧版硬编码像素偏移。

### 影响

- 角色视觉位置偏上（尤其 110px 体型）
- 气泡箭头无法准确指向嘴巴
- 眼球追踪偏移量在边缘位置不准确
- 三种体型下表现不一致

### 根本原因

重构时未全量审查，新旧逻辑共存。`layout.js` 使用新公式但 `pixel-character.js` 沿用旧偏移。

### 修复方向

1. 全量审查 `pixel-character.js` 中坐标计算
2. 将所有 `petCX ± 硬编码值` 替换为 `petCX ± 比例 × petSize`
3. 三种体型（50/80/110）回归测试

### 教训

已在 `docs/99_归档/开发日志.md` 中记录。核心教训：重构时必须全量审查，不能让新旧逻辑共存。
