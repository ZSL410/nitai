# 常见问题

## 启动问题

**Q: 启动后窗口不显示？**
A: 检查是否在 WSL2 中运行。WSL2 不支持 GUI，请在 Windows 原生命令行中运行 `npm start`。

**Q: 提示 `electron: command not found`？**
A: 依赖未安装。运行 `cd D:\nitai\拟态开发版 && npm install`。

**Q: `npm start` 报错？**
A: 确保 Node.js ≥ 18 且 `node_modules/` 完整。如有问题尝试删除 `node_modules/` 后重新 `npm install`。

## 显示问题

**Q: 角色偶尔跑到屏幕左上角卡住？** 🐛
A: 已知问题。与边界检测和路径规划逻辑有关。角色在移动到屏幕边界附近时可能走到 (0,0) 附近。临时处理：拖拽角色到新位置即可恢复。

**Q: 眼球追踪有抖动？**
A: 一阶 lerp 平滑（速率 0.28）在快速移动时有轻微抖动。影响轻微，优先度低。

**Q: 角色视觉位置偏上？**
A: v1.0.0 已修复此问题。角色现在始终在窗口中央渲染（`character/pixel-renderer.js` 的 `getCharacterBounds()` 中以 `winW/2`, `winH/2` 为中心）。

## 功能问题

**Q: 文件拖放没反应？**
A: 检查：
1. `config.json` 中 `targetDir` 路径是否存在
2. 拖放的文件是否可读
3. DevTools Console 中是否有报错

**Q: 听歌模式不自动触发？**
A: 检查：
1. `config.json` 中 `musicDetect` 是否为 `true`
2. 播放器名称是否在 `musicPlayers` 列表中
3. 播放器进程名是否与配置一致

**Q: 手动触发听歌后如何退出？**
A: 按 Enter → 输入 "停止" 或 "stop"。

**Q: 穿透模式下无法操作桌宠？**
A: `Ctrl+Shift+P` 全局快捷键在穿透模式下仍然可用。也可通过系统托盘菜单关闭穿透。

**Q: Ctrl+Q 无法退出？**
A: 窗口需要有焦点才能接收键盘事件。如果窗口失去焦点，使用系统托盘菜单退出。

**Q: 角色不自动闲逛了？**
A: 检查：`Ctrl+W` 或右键菜单 → "🚶 开始闲逛" 可以切换闲逛开关。另外，角色在 walking/curious/happy/surprised 等非 idle 状态下不会做自主决策，等待回到 idle 即可。

**Q: 拖拽后角色位置异常？**
A: v1.0.0 中拖拽会同步更新世界坐标。如果出现不同步，等待 5 秒自动同步周期即可恢复。
