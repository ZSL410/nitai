# 拟态桌宠（Mimic Desktop Pet）— 功能手册 v3.7.1

> 最后更新：2026-06-27 | 版本：**v3.7.1**

拟态桌宠是晓风 Agent 体系的 Windows Electron 桌面宠物，以 16×20 网格像素小人呈现，具备肢体动画、表情系统、音效合成、粒子特效和丰富的交互能力。

## 交互方式

### 鼠标交互

| 操作 | 效果 | 说明 |
|------|------|------|
| **眼球跟随** | 瞳孔追踪鼠标位置 | idle 状态下，瞳孔基于头部中心计算偏移，范围限制在头部区域内（±2×±1 格）。鼠标离开窗口时瞳孔平滑回到中心。 |
| **点击头部** | `happy` 状态 | 弹跳 + 微笑（arch 眼型）+ chirp 音效 + 心形粒子，**1.5 秒后自动恢复 idle** |
| **点击身体** | `surprised` 状态 | 眼睛睁大（wide 眼型）+ 嘴巴张开 + boing 音效 + 火花粒子，**1.5 秒后自动恢复 idle** |
| **双击** | 旋转弹跳 | 火花粒子 + boing 音效 + 2.5s happy 状态 |
| **拖拽** | 移动窗口 | Pointer Capture 拖拽（v2.1.4 WSL2 兼容） |

### 键盘交互

### 聊天命令

| 输入 | 触发 |
|------|------|
| `bongo` | Bongo Cat 敲击彩蛋 |
| `听歌` / `music` / `耳机` | 听歌模式（戴耳机 + 闭眼摇摆 + 音符飘出） |

| 其他文本 | Mock 回复（无后端时）或 POST `/api/chat` |

## 状态机（FSM）

共 8 个状态，由 `fsm/fsm-core.js` 引擎驱动：

| 状态 | 触发条件 | 动画 | 表情 | 时长 |
|------|----------|------|------|------|
| **idle** | 默认 | 呼吸起伏 + 眨眼 | neutral | 常驻 |
| **working** | 文件拖放 | 张嘴吃文件（nom 节奏 ~400ms） | working | 直至完成 |
| **happy** | 点击头部 / 双击 | 弹跳 + 双臂挥舞 | happy（arch + smile + cheek） | 1.5s / 2.5s |
| **surprised** | 点击身体 | 抖动衰减 + 双臂张开 | surprised（wide + open） | 1.5s |
| **alert** | （预留） | 蹲下蓄力 → 跳起 + 单臂指天 | alert（sparkle） | 常驻 |
| **bongo** | 输入"bongo" | 坐姿 + 双手交替拍击 | bongo（arch + grin + cheek） | 10s |
| **listening** | 输入"听歌" / 音乐检测 | 戴耳机 + 闭眼摇摆 | listening（closed_arch + cheek） | 手动退出 |
| **sleepy/sleeping** | idle 子状态 | 60s→sleepy, 120s→sleeping（Zzz 气泡） | sleepy/sleeping | 鼠标唤醒 |

## 低功耗空闲模式 🆕 (v3.7.1)

- 角色处于 `idle` 状态且**连续 30 秒无鼠标移动**时，绘制帧率降至 **15fps**
- 帧跳过机制：每 4 帧绘制 1 帧（60÷15=4）
- 鼠标移动（`mousemove` 事件）**立即**恢复 60fps 全速绘制
- 切换体型时 `mousemove` 事件自动重置空闲计时器
- 目的：降低 CPU/GPU 占用，适合长时间桌面驻留

## 眼球追踪 🆕 (v3.7.1 增强)

- 基于**角色头部中心**（而非窗口中心）计算鼠标偏移
- 眼珠偏移范围：水平 ±1.5 格、垂直 ±1.0 格（严格限制在头部 cols 4-11 / rows 0-6 区域内）
- 渲染层支持 ±2×±1 格的像素位移
- 平滑跟随（lerp 速率 0.28），避免抖动
- 仅在 `idle` 状态下激活；非 idle 或鼠标离开 → 瞳孔平滑归中

## 文件拖放

拖放文件/文件夹到桌宠窗口 → 递归复制到 `D:\xiaofeng_agent\02_数据输入`（或 `config.json` 中配置的 `targetDir`）。复制过程中显示：

- **working 状态**：张嘴吃文件动画
- **进度条**：`nom-progress` 气泡
- **后端通知**：POST `/api/file/process`

## 音效系统

Web Audio API 合成（无需外部文件）：

| 音效 | 触发场景 |
|------|----------|
| chirp | happy（点击头部/双击） |
| boing | surprised（点击身体）/ 双击 |
| yawn | 打哈欠 / 伸懒腰 |
| nom | 进食（文件复制） |
| bongo | Bongo Cat 敲击 |

## 粒子特效

| 类型 | 形状 | 触发场景 |
|------|------|----------|
| heart | ❤ | happy 状态 |
| star | ✦ | alert / bongo |
| spark | · | 双击 / 身体点击 |
| eat | 碎屑 | working（进食欲） |
| note | ♪ | bongo 敲击 / listening / idle 哼唱 |
| rain | 💧 | idle 超 10s 偶尔下落 |

## 位置记忆

窗口移动/关闭时自动保存位置到 `config/position.json`，启动时恢复。

## 配置

`config.json` 关键字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `targetDir` | string | 文件拖放目标目录 |
| `backendUrl` | string | 后端 API 地址（默认 `http://127.0.0.1:5001`） |
| `apiEnabled` | bool | 是否启用后端 API |
| `passthrough` | bool | 启动时是否默认穿透 |
| `musicDetect` | bool | 是否启用音乐自动检测 |
| `musicCheckInterval` | number | 音乐检测轮询间隔（秒） |
| `musicPlayers` | string[] | 监控的播放器进程名列表 |
