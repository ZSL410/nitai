## 三、关键决策记录

### 3.1 技术选型

| 决策 | 日期 | 原因 |
|------|------|------|
| 选择 Electron 而非 C# WinForms | 2026-06 | 跨平台潜力；Web 技术栈（Canvas/CSS）更适合像素渲染和动画；与晓风 Node.js 后端技术栈统一 |
| Canvas 2D 而非 WebGL | 2026-06 | 16×20 像素网格不需要 GPU 渲染；Canvas 2D 像素控制更精确；代码更简单 |
| FSM 状态机而非行为树 | 2026-06 | 桌宠状态数量有限（~8 个）；FSM 逻辑直观，状态切换条件清晰；代码量小 |
| Web Audio 合成而非音频文件 | 2026-06 | 无外部依赖，分发简单；合成音效符合像素风格；体积小 |
| 项目从晓风主仓库分离 | 2026-06 | 独立版本管理；降低晓风仓库复杂度；桌宠功能已足够独立成项目 |

### 3.2 架构决策

| 决策 | 原因 |
|------|------|
| 全局命名空间 `window.Mimic` 而非 ES Module | Electron 渲染进程直接加载 script 标签，无打包工具；全局命名空间简单直观 |
| 按依赖顺序手动加载脚本（index.html） | 明确的依赖链，无循环依赖；不需要模块打包器 |
| 布局公式：`winH = petSize + 2×max(overhead, BOTTOM_MARGIN)` | 保证角色绝对居中 + 气泡完整可见 + 底部留白 |
| IPC 扁平参数传递 | 绕过 Electron 上下文隔离的对象序列化边缘情况 |

### 3.3 听歌模式设计

听歌模式是拟态最"有温度"的功能。设计思路：
- **手动触发**：用户在聊天框输入"听歌"→ 进入 listening 状态（戴耳机 + 摇摆 + 音符飘出）
- **自动检测**：后台 3 秒扫描进程列表，检测到 Spotify/QQ音乐/网易云 → 自动触发
- **自动退出**：自动触发的 listening 在音乐停止后自动退出；手动触发的不受影响
- **视觉**：耳机（头梁 + 耳罩）+ closed_arch 眼型 + 腮红 + 身体摇摆 + 音符粒子

## 四、犯过的错误汇总

### 4.1 在 WSL 中运行桌宠

**错误**：试图在 WSL2 中运行 Electron GUI 应用。

**后果**：窗口无法创建，`--no-sandbox --disable-gpu` 参数只能勉强运行但极其不稳定。

**教训**：WSL2 仅用于代码编辑和 Git 操作。桌宠的开发和运行必须在 Windows 原生环境进行。`npm run start:wsl` 仅作为紧急调试手段保留。

### 4.2 坐标硬编码

**错误**：在 v2.0 架构重写时，布局公式设计为动态计算（基于 petCX/petCY），但 `pixel-character.js` 中的肢体动画端点仍使用硬编码像素偏移。

**后果**：
- 角色在窗口中的视觉位置偏上
- 气泡箭头无法准确指向嘴巴（偏差随体型放大）
- 三种体型（50/80/110）下表现不一致

**教训**：
- 重构时必须全量审查，不能让新旧逻辑共存
- 所有坐标计算必须以 `petCX`/`petCY`/`petSize` 为基准
- 应在三种体型下做回归测试

**修复状态**：待修复（计划 v3.8.0 / 独立后 v1.0.0）

### 4.3 版本号混乱

**错误**：版本号在 `app.js`（v3.7.1）、`mimic.js`（v2.1.0）、`main.js`（v2.5.0）三处独立定义且不同步。

**后果**：`console.log` 输出矛盾版本号，调试时无法确定实际运行版本。

**教训**：
- 版本号应有单一来源
- 每次发版必须同步更新所有定义处
- 已在 CLAUDE.md 中记录同步规则

**修复状态**：已修复（v2.1.1），但需持续遵守

### 4.4 IPC 对象参数序列化崩溃

**错误**：IPC 通道使用对象参数 `send('channel', {dx, dy})`。

**后果**：`TypeError: conversion failure` 崩溃（v2.1.2 - v2.1.4 期间反复出现）。

**教训**：
- Electron 上下文隔离下对象序列化存在边缘情况
- 改用扁平参数 `send('channel', dx, dy)` + 主进程 `Number()` 守卫
- 所有 IPC 通道统一此模式

**修复状态**：已修复（v2.1.4）

### 4.5 Pointer Capture 在 WSL2 不兼容

**错误**：使用 `setPointerCapture` 实现拖拽。

**后果**：WSL2 下 `conversion failure` 崩溃。

**教训**：改用 document 级 mousemove 追踪 + try-catch 包裹。

**修复状态**：已修复（v2.1.4）
