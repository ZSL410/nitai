# 拟态（nitai）项目说明书

> 版本：v3.7.1 | 最后更新：2026-06-30 | 作者：ZSL410

---

## 一、项目起源与定位

### 1.1 起源

拟态（nitai）诞生于晓风 Agent 体系对"桌面存在感"的需求。晓风是一个运行在后台的智能助手，用户需要一种直观的方式感知它的状态 — 它在工作吗？它收到了我的文件吗？它不开心吗？

答案是：在屏幕右下角养一只像素小人。

### 1.2 定位

- **晓风的数据入口**：用户拖放文件到桌宠 → 自动复制到晓风数据目录 → 通知后端处理
- **晓风的情绪出口**：后端可以推送状态到桌宠，让它做出对应表情和动画
- **用户的桌面陪伴**：眼球追踪、点击反应、对话互动 — 不只是工具，是住在屏幕里的小家伙

### 1.3 命名

"拟态"一词取自动物学中的 mimicry（拟态）— 生物模拟环境中其他生物以融入环境。桌宠"拟态"成一个桌面小生物，融入你的工作环境。

### 2.2 主进程（main/）职责

| 模块 | 职责 |
|------|------|
| `config.js` | 加载 `config.json`，提供 `getConfig()` 接口 |
| `window.js` | 创建透明无边框窗口；管理 `setIgnoreMouseEvents` 穿透模式；提供 `getWindow()` / `setPassthrough()` / `isPassthrough()` |
| `tray.js` | 系统托盘图标 + 菜单（体型切换/穿透开关/退出）；`refreshTrayMenu()` 同步穿透状态 |
| `ipc-handlers.js` | 注册所有 IPC 通道：`window-drag` / `resize-window` / `set-pet-size` / `quit-app` / `toggle-passthrough` |
| `music-detector.js` | 每 3 秒执行 `tasklist`（Windows）扫描进程列表，匹配配置的播放器名，发送 `music-playing` / `music-stopped` IPC |
| `position-store.js` | 窗口关闭时保存位置到 `config/position.json`，启动时读取恢复 |

### 2.3 渲染进程（renderer/）职责

| 模块 | 职责 |
|------|------|
| `index.html` | HTML 骨架 + CSS 样式 + 按依赖顺序加载所有 JS 模块 |
| `app.js` | 引导入口：设置版本号 → 绑定 DOM → 调用各模块初始化 → 启动 rAF 动画循环 + 低功耗帧率控制 |
| `mimic.js` | 全局命名空间 `window.Mimic`：共享状态变量 + 布局常量 + IPC/fs/path/http 引用 |
| `layout.js` | 动态布局：测量气泡宽度 → 计算窗口尺寸 → 角色居中 → 通知主进程 resize |
| `audio.js` | Web Audio API 合成五种音效，懒初始化 AudioContext，失败静默降级 |
| `fsm/fsm-core.js` | FSM 引擎：状态切换/生命周期钩子/表情查询 |
| `fsm/expressions.js` | 表情定义表：11 种表情（眼型 + 嘴型 + 腮红） |
| `fsm/states/*.js` | 每个状态的 enter / update / exit 逻辑 |
| `rendering/pixel-character.js` | 16×20 网格像素小人完整绘制（头/身/四肢/头发/脚/耳机/眼型/嘴型） |
| `rendering/particles.js` | 6 种粒子特效 + 物理模拟（飘升/漂移/淡出） |
| `bubble/bubble.js` | 消息队列（多条消息顺序显示）+ 嘴边定位（箭头指向嘴巴） |
| `interaction/*.js` | 拖拽/右键/拖放/眼球/点击/对话/音乐检测 |

## 三、FSM 状态机

### 3.1 引擎设计

状态机由 `fsm/fsm-core.js` 实现，核心 API：

```javascript
M.FSM.transitionTo('stateName', { context });  // 切换状态
M.FSM.update(performance.now());                // 每帧更新
M.FSM.getExpression();                          // 获取当前表情
```

状态切换自动调用生命周期钩子：

```
_exit(oldState) → _enter(newState) → _update(newState, now)  // 每帧
```

### 3.2 状态定义

| 状态 | 触发条件 | 动画描述 | 表情 | 持续时间 |
|------|----------|----------|------|----------|
| **idle** | 默认状态 | 呼吸起伏 + 眨眼 + 眼球追踪；30s 无操作→打哈欠；60s→sleepy；120s→sleeping（Zzz 气泡）| neutral / sleepy / sleeping | 常驻 |
| **working** | 文件拖放 | 张嘴吃文件（nom 节奏 ~400ms）+ 碎屑飞向嘴巴 + 手臂伸手 | working（round + none）| 直至复制完成 |
| **happy** | 点击头部 / 双击 | 弹跳 + 双臂举起挥舞 + chirp 音效 + 心形粒子 | happy（arch + smile + cheek）| 1.5s / 2.5s |
| **surprised** | 点击身体 | 后退 + 双臂张开 + 抖动衰减 + boing 音效 + 火花粒子 | surprised（wide + open）| 1.5s |
| **alert** | （预留） | 蹲下蓄力 → 跳起 + 单臂指天 + 星星眼 + 星形粒子 | alert（sparkle + none）| 常驻 |
| **bongo** | 输入"bongo" | 坐姿 + 双手交替拍击 + bongo 音效 + 音符粒子 | bongo（arch + grin + cheek）| 10s |
| **listening** | 输入"听歌"/自动检测 | 戴耳机 + 闭眼（closed_arch）+ 身体摇摆 + 每 600ms 音符飘出 | listening（closed_arch + none + cheek）| 手动退出/音乐停止 |
| **sleeping** | idle 子状态 120s | Zzz 气泡 + 闭眼；鼠标移动 → 伸懒腰（yawn 音效）→ idle | sleeping（closed + none）| 鼠标唤醒 |

### 3.3 扩展新状态

1. 创建 `fsm/states/<name>.js`，导出 `{ expression, enter, update, exit }`
2. 在 `fsm/expressions.js` 中添加对应表情定义
3. 在 `index.html` 中按依赖顺序添加 `<script>` 标签

## 四、表情系统

### 4.1 表情定义表

表情是**数据驱动**的，每个表情由一个眼型、一个嘴型和一个腮红开关组成：

| 表情名 | 中文 | 眼型 (eyes) | 嘴型 (mouth) | 腮红 (cheek) | 使用状态 |
|--------|------|-------------|-------------|-------------|----------|
| neutral | 平静 | round | none | false | idle |
| happy | 开心 | arch | smile | true | happy |
| working | 专注 | round | none | false | working |
| open | 张嘴 | round | open | false | working（进食瞬间）|
| surprised | 惊讶 | wide | open | false | surprised |
| sleepy | 困倦 | half_closed | none | false | idle（60s）|
| proud | 骄傲 | closed_arch | grin | true | （预留）|
| alert | 提醒 | sparkle | none | false | alert |
| sleeping | 睡觉 | closed | none | false | idle（120s）|
| bongo | 敲击 | arch | grin | true | bongo |
| listening | 听歌 | closed_arch | none | true | listening |

### 4.2 眼型渲染（pixel-character.js）

| 眼型 | 像素绘制 | 说明 |
|------|----------|------|
| `round` | 2×2 方块 | 标准圆眼 |
| `arch` | 2×2 方块上移 1px | 开心弧形眼 |
| `wide` | 3×2 方块 | 惊讶大眼 |
| `closed` | 1px 高直线 | 闭眼 |

| `none` | 不绘制 |

## 五、交互模块详解

### 5.1 拖拽移动（drag.js）

- 使用 Pointer Capture 机制（WSL2 兼容模式下退化为 document 级 mousemove 追踪）
- 移动时通知主进程 `window-drag` IPC（传递 dx/dy 增量）
- 移动结束保存位置到 `config/position.json`
- `mousemove` 事件同时重置低功耗空闲计时器

### 5.2 文件拖放（drop.js）

- 监听 `dragover` / `dragleave` / `drop` 事件
- `dragover` 时显示虚线高亮遮罩（#overlay）
- `drop` 时递归复制文件/文件夹到 `targetDir`
- 复制过程触发 `working` 状态 + 进度条气泡 + nom 音效
- 复制完成发送 `POST /api/file/process` 通知后端
- 使用 `webUtils.getPathForFile()` 获取文件路径（Electron 安全限制）

### 5.3 眼球追踪（eye-tracking.js）

- 仅在 `idle` 状态下激活
- 基于**角色头部中心**（而非窗口中心）计算鼠标偏移量
- 瞳孔偏移范围：水平 ±2 格、垂直 ±1 格（严格限制在头部区域内）
- 平滑跟随：lerp 速率 0.28
- 鼠标离开窗口时瞳孔平滑回到中心

### 5.4 点击反应（click.js）

| 操作 | 触发状态 | 音效 | 粒子 | 持续时间 |
|------|----------|------|------|----------|
| 点击头部 | happy | chirp | 心形 ❤ | 1.5s |
| 点击身体 | surprised | boing | 火花 · | 1.5s |
| 双击 | happy（旋转弹跳）| boing | 火花 · | 2.5s |

- 点击部位判定基于角色身体区域的网格坐标
- 音效和粒子在状态切换 `context` 中传递触发

### 5.5 对话交互（chat.js）

- 按 `Enter` 弹出输入框（透明背景浮层）
- 输入文本后：
  - 匹配彩蛋关键词：`bongo` → bongo 状态 / `听歌`/`music`/`耳机` → listening
  - 匹配退出关键词：`停止`/`stop` → 退出 listening/bongo
  - `apiEnabled=true` → POST `/api/chat` 获取后端回复
  - `apiEnabled=false` → 本地 Mock 中文关键词匹配
- 回复显示在气泡中

### 5.6 穿透模式

- 三种切换方式：系统托盘菜单 / 右键菜单 / `Ctrl+Shift+P`
- 穿透状态下鼠标事件直达下层窗口（`setIgnoreMouseEvents(true, {forward: true})`）
- 全局快捷键在主进程注册，确保穿透模式下仍可切回

### 5.7 低功耗空闲模式（v3.7.1）

- 角色处于 `idle` 且连续 30 秒无鼠标移动 → 帧率降至 15fps
- 实现方式：帧跳过计数器（每 4 帧绘制 1 帧）
- 鼠标移动立即恢复 60fps
- 切换体型时 `mousemove` 事件自动重置计时器

## 六、气泡系统

### 6.1 架构

- **消息队列**：`bubble.js` 维护一个队列，多条消息按顺序显示，每条间隔 2 秒
- **嘴边定位**：气泡从角色嘴巴上方"长"出，带 CSS `::after` 伪元素三角箭头指向嘴巴
- **自适应尺寸**：内容增加时气泡变宽（最大窗口宽度的 80%），换行后高度增加

### 6.2 气泡类型

| 类型 | 说明 |
|------|------|
| 普通消息 | 白色圆角矩形 + 箭头尾巴 |
| 进度条 | 文件复制时显示 `nom-progress` 进度 |
| 思考气泡 | "…" 动画（chat 模式等待回复时）|
| Zzz 气泡 | sleeping 状态偶发显示 |

### 6.3 与窗口布局的联动

气泡出现/更新/消失时，都会调用 `Layout.updateWindowSizeAndLayout()` 重新计算窗口尺寸：

```
overhead = bubbleH + ARROW_H(6px) + GAP(5px)
winH = petSize + 2 × max(overhead, BOTTOM_MARGIN(10px))
```

窗口高度动态调整，确保气泡完整可见 + 角色居中。

## 七、与后端 API 交互

### 7.1 配置

`config.json` 中的后端配置：

```json
{
  "backendUrl": "http://127.0.0.1:5001",
  "apiEnabled": false,
  "targetDir": "../02_数据输入"
}
```

### 7.2 接口

#### POST /api/file/process

文件复制完成后调用，通知晓风后端处理新文件。

```json
{
  "path": "D:\\xiaofeng_agent\\02_数据输入\\example.pdf",
  "filename": "example.pdf"
}
```

#### POST /api/chat

对话消息发送到后端，获取 AI 回复。

```json
// 请求
{ "message": "今天天气怎么样？" }

// 响应
{ "reply": "让我想想...今天阳光很好！" }
```

无后端时（`apiEnabled: false`），chat.js 使用本地 Mock 中文关键词匹配。

## 八、已知问题与修复方案

### 8.1 坐标硬编码残留

**严重程度**：中

**表现**：部分肢体动画端点使用硬编码像素偏移，在非默认体型（80px）下角色位置偏上，气泡箭头无法准确指向嘴巴。

**根因**：v2.0 重写时引入的核心布局公式（`petCX=winW/2, petCY=winH/2`）未渗透到所有绘制逻辑中，`pixel-character.js` 中部分肢体端点仍使用旧版的固定偏移。

**修复方案**：
1. 全量审查 `pixel-character.js` 中的坐标计算
2. 将所有 `petCX ± 固定值` 替换为 `petCX ± 比例值 × petSize`
3. 在三种体型（50/80/110）下验证角色居中 + 气泡对齐

**计划版本**：v3.8.0

### 8.2 眼球追踪卡顿

**严重程度**：低

**表现**：快速移动鼠标时眼球过渡有轻微视觉抖动。

**根因**：一阶 lerp 平滑（速率 0.28）在低帧率或大幅度移动时响应不够平滑。

**修复方案**：
1. 尝试二阶平滑（加速度 lerp）
2. 或提高低功耗模式的检测灵敏度，避免 lerp 在高/低频间切换

**计划版本**：v3.8.x

### 8.3 版本号不一致

**严重程度**：已修复

在 v2.1.1 之前，版本号在 `app.js`（3.7.1）、`mimic.js`（2.1.0）和 `main.js`（2.5.0）中分别定义且不同步。现已统一但需在每次发版时手动同步三处。建议引入单一版本号源。

### 8.4 WSL2 兼容性问题

**严重程度**：已知限制

WSL2 无法运行 GUI，`npm run start:wsl` 使用 `--no-sandbox --disable-gpu` 参数尝试兼容但稳定性无法保证。桌宠开发/运行应在 Windows 原生环境进行。

## 九、配置参考

| 字段 | 类型 | 默认值 | 说明 |
