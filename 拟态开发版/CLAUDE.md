# 项目：拟态桌宠 (Windows Electron)

## 核心规则（必须遵守）

### 1. 优先读取项目状态
在开始任何代码修改或诊断之前，你**必须**先读取本目录下的 `PROJECT_STATUS.md` 文件。该文件记录了项目的当前版本、已完成功能、已知问题（尤其是布局、居中、气泡遮挡问题）以及下一步计划。所有决策必须基于该文档的最新信息。

### 2. 版本号管理
- 每次代码修改（优化、修复、功能增加）都必须递增版本号。
- 版本号格式：`主.次.补丁`（例如 `1.1.0` → `1.1.1` 或 `1.2.0`）。
- 在 `renderer/app.js` 顶部定义 `const APP_VERSION = 'x.x.x';`。
- 同步更新 `PROJECT_STATUS.md` 中的版本号和变更日志。

### 3. 窗口与角色布局（最终目标）
- **窗口随内容动态调整**：窗口宽度 = 角色宽度 + 40px（左右各 20px 留白）。窗口高度由数学公式确定以保证角色绝对居中。
  - 实际公式（v1.2.0）：`winH = petSize + 2 × max(overhead, BOTTOM_MARGIN)`
  - 其中 `overhead = bubbleH + ARROW_H + GAP`（气泡总开销），`BOTTOM_MARGIN = 10`
  - 这确保 petTop ≥ overhead（气泡完整可见）且底部留白 ≥ BOTTOM_MARGIN
  - 简化近似公式（当 overhead ≥ BOTTOM_MARGIN 时）：`winH ≈ petSize + 2×(bubbleH + ARROW_H + GAP)`
- **角色绝对居中**：角色的中心必须与窗口的几何中心完全重合（`petCX = winW/2, petCY = winH/2`）。
- **气泡位置**：气泡底部（箭头尖端）紧贴角色顶部（间隙 5px），水平居中于角色。气泡 CSS `top = petTop - GAP - bubbleH`。
- **气泡自适应**：内容增多时气泡可变宽（最大窗口宽度的 80%），换行后高度增加，窗口同步增高。

### 4. 自主升级流程
- 当你收到“自我升级”或“修复布局问题”的指令时，请按照以下步骤执行：
  1. 先读取 `PROJECT_STATUS.md`。
  2. 使用 MCP 服务器 `skills-master` 搜索并安装与 Electron 窗口布局、居中、动态布局相关的社区技能。
  3. 应用这些技能，系统性地重构 `renderer/app.js`，一次性解决角色居中和气泡遮挡问题。
  4. 修改完成后，递增版本号，并更新 `PROJECT_STATUS.md`。
